Content Density can be maintained app-wide using the ContentDensityService, and components can subscribe to a BehaviorSubject managed by the service and use that to handle changes to the content density. Before using it, you should add ContentDensityService to the providers array of AppModule, or other root modules in your application.
Note that many components only come in one size, and others (such as the button below) only have compact and cozy sizes, so setting the size to condensed will have the same effect as compact.
All components with content density options will check to see if the developer has provided a compact input value. If not, it will then refer to the value set in the ContentDensityService.
